# ⚗️ モヤモヤの錬金術師 — Edward & Alphonse

**QQ #032 / #033 — Quest Queries Series**

---

## これは何か

「アプリを作りたい。でも、何から始めればいいかわからない。」

そのモヤモヤを持つ人のための、**2段階の錬成ツール**です。

頭の中でフローチャートが書けなくても大丈夫。  
AIに「全部やって」と丸投げするのが物足りなくても大丈夫。  
エドワードとアルフォンスが、**一緒に考えながら**連れていきます。

---

## 2人の兄弟

### ⚗️ Edward \`#032\` — モヤモヤの錬金術師

対話を通じて、漠然とした「作りたい何か」を  
**アプリの種**という形に結晶化させます。

壮大な要件が出てきたら「まず一つに絞ろう」と誘導し、  
「面倒なこと」「困っていること」を丁寧に言語化していく。  
難しく考えなくていい。声に出すつもりで、ざっくりと。

### 📜 Alphonse \`#033\` — 要件定義の書記官

エドワードから受け取った種を復唱し、  
色調・使用場面・時間帯・トーンを**一緒に選びながら**  
要件定義書とREADMEを生成します。

「あなたのアプリをどんな姿にしたいですか？」  
パネルをポチポチ選ぶだけで、仕様が決まっていく。  
答えだけ渡すのではなく、**自分が作ったという感覚**を大切にしています。

---

## 使いかた

```
1. index.html を開く
2. Edward でモヤモヤを話す（3〜4ターン）
3. 「アルフォンスに渡す」ボタンを押す
4. Alphonse でUI仕様を選んで生成
5. 要件定義書・READMEを保存する
   └ これで「何を作るか」が言語化された状態になります
6. あとは Claude / Claude Code にこの仕様書を渡して
   アプリ本体を作ってもらいましょう```
---
## なぜアルフォンスはコードを書かないのか

実を言えば、アルフォンスにはアプリのコードそのものを錬成する力もある。

しかし——錬金術には、等価交換の原則がある。  
コードの錬成は出力が大きくなりすぎ、世界の調和を乱す。  
今はまだ、**定義までが、アルフォンスの領分。**

この先どうするか——それはまた、みんなで考えよう。

## 技術構成

- **Single HTML** — Vanilla JS、フレームワーク不使用
- **Anthropic Claude API** — `claude-sonnet-4-20250514`
- **localStorage** — APIキー保存 & 兄弟間のデータ受け渡し（キー: `ed_to_al_seed`）
- **GitHub Pages** — デプロイ先

APIキーはブラウザのlocalStorageに保存されます。外部サーバーには送信しません。

## このプロジェクトが大切にしていること

**「結晶化」という体験**

モヤモヤしたアイデアが、対話を通じて輪郭を持ち始める瞬間がある。  
その瞬間こそが、このツールが届けたいものです。

答えだけ渡すAIは速い。でも、**プロセスに価値がある**。  
自分で選んで、自分で確かめて、「これが私のアプリだ」と思える——  
そこまで連れていくのが、エドワードとアルフォンスの仕事です。
